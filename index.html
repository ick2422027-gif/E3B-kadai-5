仮想センサによる連続ループ演算と常時監視ロジックの構築

WBGT担当：山崎 勇気（第4回経過報告）

実機センサの配線・取得テストが進む間の「ソフトウェア先行開発」として、マイコン内での常時監視を想定したプログラムへと改善しました。1秒周期でデータを自動取得して計算し続ける「連続ループ処理」を実装。さらに、実機の代わりに5〜35℃の気温、20〜90%の湿度データを自動生成する「仮想センサコード（ダミーデータ発生環境）」を構築し、広範囲な変動でもシステムがクラッシュせず、リアルタイムにWBGT値を算出して3段階の危険度フラグを動的に切り替えられる追従性を実証しました。

ドットマトリクスLEDによる安全・危険の視覚化表示

LED担当：藤原 海音 / 鈴木 敬吾（第4回経過報告）

7セグメントLEDでの数値出力実験を経て、発光数をさらに増やしたドットマトリクスLEDを用いた絵文字（笑顔など）の出力に挑戦しました。熱中症の危険度を視覚的に判別できるようにするための基礎研究です。プログラム内でビットパターンデータを定義し、右ビットシフト演算（>>）と論理積（& 0x01）を組み合わせた型抜き処理（ビット判定ロジック）を実装。8マスのうち光らせたいマスの情報だけを一瞬の計算でピンポイントに抽出・制御することに成功しました。

実環境データによるWBGTアルゴリズム検証と精度テスト

WBGT担当：山崎 勇気（第3回経過報告）

実際の環境（気温25.1℃ / 湿度63%）において、自作プログラムによるWBGT計算値と、信頼性の高い外部WEBツールの計算結果を比較テストしました。複雑な逆三角関数（arctan）を用いたアルゴリズムが正確に機能しているかを検証した結果、数値は「誤差なしの完全一致」を達成。あわせて、WBGT値の約7割が「湿度」に依存するという熱ストレスの特性（日本の夏の不快リスク）を再確認し、If構文を用いた3段階（緑・黄・赤）の危険度判定ロジックが正常に動作することを確認しました。

7セグメントLEDを用いた数値出力と共通設計図の構築

LED担当：藤原 海音 / 鈴木 敬吾（第3回経過報告）

バーサライタでの将来的な文字出力を見据え、目視で情報がはっきり分かるように7セグメントLEDを用いた0〜9のカウントアップ回路を設計しました。前回のdelay()関数によるミリ秒単位の速度制御を応用したほか、使用する7セグの種類（カソードコモン／アノードコモン）の性質の違いに対応するため、プログラム側で自動的にHIGHとLOWの信号を反転させて翻訳する「共通設計図（自動判定ロジック if (!IS_CATHODE_COMMON)）」を組み込み、部品が変わってもそのまま動く柔軟なコードを開発しました。

温度・湿度からのWBGTリアルタイム演算プログラムの開発

WBGT担当：山崎 勇気（第2回経過報告）

専門的な測定器を必要とせず、温度センサと湿度センサから得られる生データのみから、屋内用WBGT（暑さ指数）をリアルタイムで算出する基本プログラムを構築しました。公式（WBGT = 0.7 × Tcalc + 0.3 × Temp）をコードに落とし込み、算出された数値に応じて「安全（21℃未満：緑点灯）」「警戒（21℃〜28℃：黄点灯）」「厳重警戒・危険（28℃以上：赤点灯）」へとLEDカラーを連動させる危険度判定ロジックの基盤を確立しました。

Arduinoによる3色LEDの条件分岐ウェーブ点灯制御

LED担当：藤原 海音 / 鈴木 敬吾（第2回経過報告）

3色同時点灯の成功を踏まえ、Arduino IDEを用いた本格的なプログラム制御に移行しました。変数 count を定義して時間の経過とともにループ内で加算させ、if - else if による条件分岐（count <= 10、20、30）を構築。11番・12番・13番の各ピンへの出力を順次切り替えることで、赤・黄・緑の3色のLEDが流れるようにウェーブ状に点灯する回路とアルゴリズムを完成させました。

WBGT（暑さ指数）の基本概念と屋内算出アプローチの検討

WBGT担当：山崎 勇気（第1回経過報告）

熱中症アラートシステムの根幹を支える指標として、WBGT（暑さ指数）の専門的な調査を行いました。気温・湿度・日射の3要素が人体に与える熱負担について学び、今回は「教室等の屋内」をターゲットとするため、日射の影響を除外した屋内用基本公式の採用を検討。環境省や日本生気象学会のガイドラインを基準に、測定器なしで温度と湿度から数値を割り出すための理論的アプローチと、班内での役割分担を策定しました。

温度センサー（TMP36）の特性理解とシミュレーション検証

センサー担当：鈴木 暖人 / 増田 淑紀（第1回経過報告）

熱中症検知の頭脳となる温度センサー（TMP36）のデータシートを読み解き、温度変化に伴う内部抵抗の変化（0℃の時に500mV出力、1℃上昇ごとに10mV増加する特性）について学びました。その後、Tinkercadを用いたオンラインシミュレーション環境を構築し、センサーとマルチメーターを接続。任意の温度設定における出力電圧を測定し、データシートの理想特性曲線と一致するかを比較・検証しました。

LEDの前提知識習得と3色同時点灯の基礎実験

LED担当：藤原 海音 / 鈴木 敬吾（第1回経過報告）

電子工作の第一歩として、LEDの極性（足の長いアノードがプラス、短いカソードがマイナス）や、データシートで定格電流（一般的には20mA）を確認する重要性を学習しました。その後、Arduino Uno、ブレッドボード、330Ωの抵抗3個を使用し、赤・黄・緑の3つのLEDを過電流なく安全に同時点灯させる基本回路をブレッドボード上に構築しました。
